# Studio Vybe Image Optimizer

A native macOS app for batch image optimization — built with SwiftUI + Swift Package
Manager (no Xcode project required). Optimize JPEG / PNG / WebP / AVIF by quality or to a
target file size, per folder, with optional resize, rename, folder-watching, and
drag-onto-dock processing. Fully local — no network, no accounts, no telemetry.

## Features

- **Two compression modes**
  - **Quality** — single quality slider mapped sensibly per format.
  - **Target size** — set a max file size per format; the app binary-searches encoder
    quality (and PNG colour count) to land just under each budget, and can auto-downscale
    dimensions when quality alone can't get there.
- **Per-folder jobs** — each folder you add gets its own compression mode, per-format
  targets, output formats, and resize. The Settings tab holds the defaults new folders inherit.
- **Output format selection** — tick any of JPEG/PNG/WebP/AVIF to emit that format for every
  image (true conversion), plus a "keep original format" option to optimize in place.
- **Resize** — longest side / width / height, proportional.
- **Rename** — prefix/suffix, find & replace, regex, sequential numbering.
- **Watch folders** — auto-optimize new images dropped into monitored folders.
- **Dock drop** — drag images or folders onto the app icon to optimize instantly.
- **Never larger** — same-format output is never bigger than the original.

Outputs are written to an `optimized/` subfolder next to the source files; originals are
never modified.

## Requirements

- macOS 14+
- Swift 6 toolchain (Xcode command line tools)

### Optional but recommended CLI encoders

The app works out of the box using macOS-native ImageIO, but these produce noticeably
smaller files (and `pngquant` is what makes PNG target-sizing effective):

```bash
brew install mozjpeg pngquant oxipng libavif
```

`cwebp` (from the `webp` formula) is used for WebP fallback if present. The app detects each
tool automatically and falls back gracefully when one is missing.

## Build & run

```bash
bash build.sh            # compile + assemble ImageOptimizer.app + ad-hoc codesign
open ImageOptimizer.app
```

During development you can also `swift build` and `swift run`. After a rebuild, kill any
running instance first so macOS launches the new binary:

```bash
pkill -f ImageOptimizer; bash build.sh && open ImageOptimizer.app
```

The app icon is committed as `Resources/AppIcon.icns`. `bash build.sh --icon` regenerates it
from a brand SVG; the raw Studio Vybe logo source is not included in this repo, so `--icon`
simply skips regeneration and keeps the committed icon.

## Project layout

```
Package.swift                 Swift Package manifest
build.sh                      Build + .app bundle + codesign  (--icon to regen the icon)
scripts/make-appicon.swift    Renders a brand SVG into AppIcon.icns
samples/sample.png            Self-made (CC0) test image
Resources/
  Info.plist                  App metadata + dock-drop document types + bundled font path
  AppIcon.icns                App icon
  Fonts/                      Bundled Inter (SIL OFL — see OFL.txt)
Sources/ImageOptimizer/
  ImageOptimizerApp.swift     @main entry, dock-drop delegate, font registration
  Theme.swift                 Colour palette + Inter font helpers
  Models/                     ImageFormat, ProcessingSettings, FolderJob, ImageFile, RenameSettings, AppState
  Services/                   ImageProcessor (encoders + target-size search), FileScanner, FolderWatcher, FontLoader
  Views/                      ContentView, FolderPickerView, SettingsView, CompressionConfigEditor, SummaryTableView, WatchView
```

See `CLAUDE.md` for deeper architecture and processing-rule notes.
